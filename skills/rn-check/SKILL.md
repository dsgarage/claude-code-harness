# React Native コードチェック

Expo + React Native / TypeScript のコードを検査し、よくあるエラーパターンを検出する。

## 実行手順

1. 対象ファイルまたはディレクトリを特定する（引数があればそのパス、なければカレントディレクトリ）
2. 以下のチェック項目を順番に検査する
3. 問題を発見したら、ファイルパス・行番号・問題内容・修正案を報告する

## チェック項目

### 1. 認証トークン参照チェック
- `token` プロパティの参照がないか検索（正しくは `accessToken`）
- `useAuthStore()` のプロパティ参照が正しいか確認
- 認証が必要な API 呼び出しで `accessToken` が渡されているか確認

### 2. Expo Router ルーティングチェック
- `router.push()` / `router.replace()` のパスが `app/` ディレクトリ構造と一致するか
- 動的ルート（`[id]`）のパラメータが正しく渡されているか
- `_layout.tsx` でスクリーン定義されていない画面への遷移がないか
- 相対パスではなく絶対パス（`/` 始まり）を使用しているか

### 3. API クライアントチェック
- fetch URL が環境変数経由で解決されているか（ハードコード URL がないか）
- エラーハンドリング（try/catch）が適切か
- 認証必須 API で `accessToken` が null チェックされているか
- レスポンスの型が `src/api/types.ts` と整合しているか

### 4. Zustand ストアチェック
- ストアの状態更新が `set()` 経由で行われているか
- `persist` ミドルウェアのキーが一意か
- コンポーネントでのセレクター使用が適切か（不要な再レンダリング防止）

### 5. パフォーマンスチェック
- `FlatList` の `keyExtractor` が設定されているか
- `ScrollView` 内に `FlatList` がネストされていないか
- `StyleSheet.create()` がコンポーネント外で定義されているか
- `useCallback` / `useMemo` が適切に使用されているか（過剰使用も警告）
- `Reanimated` のワークレット関数に `'worklet';` ディレクティブがあるか

### 6. TypeScript 型安全性
- `any` 型の使用箇所をリストアップ
- API レスポンスの型付けが適切か
- `as` キャスト（型アサーション）の濫用がないか

### 7. Stripe 決済フロー
- `clientSecret` が適切に取得・使用されているか
- Payment Sheet の初期化とプレゼンテーションの順序が正しいか
- エラー時のユーザーフィードバックがあるか

### 8. i18n チェック
- ハードコードされた日本語/英語文字列がないか（`t()` を使用すべき）
- 新しい画面で翻訳キーが `ja.ts` と `en.ts` 両方に定義されているか

## 出力フォーマット

```
## チェック結果

### エラー (修正必須)
- [ファイル:行] 問題の説明 → 修正案

### 警告 (推奨修正)
- [ファイル:行] 問題の説明 → 修正案

### OK
- チェック項目名: 問題なし
```
